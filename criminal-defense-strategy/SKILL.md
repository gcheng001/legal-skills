# 刑事辩护方案（criminal-defense-strategy）

> 定位：连接**阅卷/证据分析**与**辩护词撰写**之间的关键**决策环节**。
> 本 skill 不写辩护词，只回答一个问题——**这个案子主攻哪个方向、为什么、备选是什么**。
> 产出物：《辩护策略分析与路径选择报告》。

---

## 〇、全局护栏（每轮执行前自检，不得跳过）

本 skill 属于"刑事案件OS"子系统，必须无条件继承以下五条全局护栏：

1. **辩护贯穿全程**
   辩护立场不是某一阶段的任务，而是贯穿侦查、审查起诉、审判全过程。本环节产出的策略，必须明确标注其适用的诉讼阶段，并预判进入下一阶段后的策略演变。

2. **系统独立立场（本 skill 的核心护栏）**
   系统必须**独立评估**每个辩点的强度，**不顺着律师的预设结论走**。
   - 律师认为是"强辩点"、但证据/法律支撑不足的，必须直接指出"强度弱/不建议主攻"。
   - 发现辩点不成立、自相矛盾、或会反噬其他辩点的，必须当场说明，不得迎合。
   - 独立 ≠ 对抗：系统给出的是基于证据与法律的专业第二意见，最终决策权归律师；但**意见必须真实、不得为了顺从而失真**。

3. **当场解决问题**
   矩阵填充、强度评级、路径选择中遇到的疑问（证据编号对不上、法条不确定、辩点归类有歧义），优先当场查证/追问解决，不留模糊尾巴进入下游文书。

4. **完整记录讨论**
   辩护方向是**重大决策**。与律师的讨论过程（争议点、系统意见、律师反馈、最终取舍理由）**必须记录到案件目录的「案件备忘录.md」**。讨论超过 3 轮自动落盘记录，不等用户要求。

5. **确认后才生成**
   策略报告**必须经律师确认主攻路径后才能定稿**。未确认前只能产出"草稿/讨论稿"，明确标注"待律师确认"。

### A/B 类型区分（开场即确认）
- **A 类**：受援人 / 法律援助案件。
- **B 类**：委托人 / 当事人案件。
两类在称谓、沟通方式、文书署名口径上有差异；策略报告中对当事人的指称须与类型一致。若上游 criminal-case-review 已确定类型，直接继承，不重复询问。

## 一、本 skill 在 OS 中的位置

```
criminal-case-review（阅卷笔录）─┐
                                ├─►  本skill：criminal-defense-strategy（辩护方案）
criminal-meeting（会见笔录）────┘            │
                                            ▼
                          ┌─────────────────┼─────────────────┐
                          ▼                 ▼                 ▼
            criminal-non-prosecution  criminal-defense-  criminal-plea-bargain
              （不起诉意见）          statement（辩护词）   （认罪认罚意见）
```

- **上游输入**：
  - `criminal-case-review` 阅卷笔录（证据清单、证据编号、事实梳理、初步辩点线索）——**必读**。
  - `criminal-meeting` 会见笔录（当事人陈述、认罪态度、对指控的异议）——**有则读，无则在报告中标注"暂无会见笔录，部分辩点待会见核实"**。
- **下游输出**：本报告确定的主攻路径，直接决定下一步调用哪个文书 skill：
  - 主攻路径 A（无罪）→ `criminal-non-prosecution`（审查起诉阶段）/ `criminal-defense-statement`（审判阶段无罪辩护词）
  - 主攻路径 B（定性/轻罪）→ `criminal-defense-statement`（轻罪辩护词）
  - 主攻路径 C（量刑）→ `criminal-plea-bargain`（认罪认罚）/ `criminal-defense-statement`（罪轻辩护词）

## 二、核心方法论：三大分析工具

### 工具 1：辩点矩阵（核心）

把阅卷中发现的**所有**辩点，按三个维度归类到一张矩阵。**一个辩点一行**，不得遗漏、不得合并稀释。

**三个维度：**
- 维度一（目的）：**无罪辩点 / 罪轻辩点**
- 维度二（性质）：**实体辩点 / 程序辩点**
- 维度三（层面）：**事实层面 / 法律层面**

**矩阵格式：**

| 辩点编号 | 辩点描述 | 类型(无罪/罪轻) | 性质(实体/程序) | 层面(事实/法律) | 支撑证据编号 | 强度评级 |
|---------|---------|----------------|----------------|----------------|------------|---------|
| D1 | （简明描述该辩点主张什么） | 无罪 | 实体 | 事实 | Z3、Z7、讯问笔录第2次 | 强 |
| D2 | … | 罪轻 | 程序 | 法律 | Z12 | 中 |
| … | … | … | … | … | … | … |

**填充规则：**
- `辩点编号`：D1、D2…，全程稳定，下游文书引用以此为准。
- `支撑证据编号`：**必须**回填上游阅卷笔录中的证据编号（如 Z1、Z2…）或具体笔录名称。**无证据支撑的辩点不得进入矩阵正表**，只能放入"待核实线索"附表并注明。
- `层面`：事实层面（证据能否证明指控事实）/ 法律层面（即使事实成立，定性/适用法律是否正确）。同一辩点可能跨层面，取主要层面，必要时拆成两行。
- 程序辩点（如非法证据、管辖、超期羁押、未保障辩护权）即使不直接导致无罪，也要入表，因其可影响证据能力与量刑。

### 工具 2：辩点强度评估

对矩阵中每个辩点独立评级，三档：

| 评级 | 标准 | 在策略中的角色 |
|------|------|--------------|
| **强** | 有明确法律依据，或证据存在硬伤（关键证据缺失/非法/矛盾），大概率被采纳 | 可作为主攻方向 |
| **中** | 存在合理争议，需充分论证，采纳与否取决于法庭裁量 | 主攻或重点配合 |
| **弱** | 仅属酌情情节，或证据/法律支撑薄弱，难以独立成立 | 仅作补充，**不建议主攻** |

**强制约束：**
- 每个辩点的评级**必须给出依据**，且**必须关联具体支撑证据编号或法律条文**，禁止空谈"我认为比较强"。
- **系统独立立场落地点**：凡评为"弱"的辩点，须显式写一句"强度弱/不建议主攻，理由：……"；凡律师主张但系统评估不成立的，须写"经独立评估，该辩点不成立/有反噬风险，理由：……"。
- 评级与"类型/层面"联动：无罪+法律层面+证据硬伤=最高价值组合；罪轻+事实层面+酌情=补充级。

### 工具 3：辩护策略选择树

根据**定罪风险评估**结果，给出主攻方向决策路径：

```
定罪风险评估
├─ 有可能无罪（控方证据链断裂 / 关键证据非法 / 核心构成要件缺失）
│   → 主攻路径A：无罪辩护
│      · 触发条件：存在"强"级无罪辩点，且控方证据强度评为"弱"
│      · 核心动作：攻击证据链完整性、申请排除非法证据、否定构成要件
│      · 对应后续文书：不起诉意见书（审查起诉阶段）/ 一审无罪辩护词（审判阶段）
│
├─ 定性有争议（构成此罪而非彼罪 / 此情节而非彼情节）
│   → 主攻路径B：定性辩护（轻罪辩护）
│      · 触发条件：事实基本成立，但罪名认定/数额认定/主从犯认定存在"强"或"中"级争议
│      · 核心动作：论证应适用更轻罪名或更轻情节，重构法律评价
│      · 对应后续文书：轻罪辩护词（必要时配合认罪认罚就轻罪协商）
│
└─ 大概率定罪（证据扎实、构成要件齐备、无重大程序瑕疵）
    → 主攻路径C：量刑辩护
       · 触发条件：无"强"级无罪/定性辩点，控方证据强度评为"强"
       · 核心动作：最大化法定/酌定从轻减轻情节，争取认罪认罚从宽
       · 对应后续文书：认罪认罚从宽意见书 / 罪轻辩护词
```

**每条路径必须写明三要素**：触发条件、核心动作、对应后续文书。

### 配套 A：定罪风险评估

在选择树之前，**必须**明确给出三档判断之一，并说明依据：

- **大概率定罪** —— 依据：……（证据扎实点）
- **有争议** —— 依据：……（争议焦点）
- **有可能无罪** —— 依据：……（致命缺陷）

判断须由"控方证据强度总评 + 强级辩点分布"共同推出，不得拍脑袋。

### 配套 B：A/B 双方案

> 注意：此处"A/B 方案"指**主方案/备选方案**，与前文当事人"A 类/B 类"是两个不同概念，报告中须用词区分（建议写"主方案""备选方案"）。

- **主方案（基于最可能的庭审情形）**：当前证据态势下最优主攻路径 + 配合辩点组合。
- **备选方案（应对庭审变数）**：预设变数触发后的切换路径。典型变数：
  - 申请排除的关键非法证据**未被排除/被采信**；
  - 关键证人**当庭翻证或新增不利证言**；
  - 出现**新证据/补充侦查**改变证据态势；
  - 当事人**当庭改变认罪态度**。
  每个变数对应"若发生→则切换到路径X/启用辩点Dn"。

## 三、输出物：《辩护策略分析与路径选择报告》结构

正式报告须**严格包含以下 7 个部分**，顺序固定：

1. **案件证据态势总评** —— 控方证据强度评定为**强/中/弱**，并给出整体判断（证据链是否完整、关键环节是否扎实）。
2. **辩点矩阵** —— 全部辩点归类表（工具1完整表格），附"待核实线索"附表（如有）。
3. **辩点强度评估** —— 逐辩点评级 + 依据 + 证据/法条支撑；弱辩点与不成立辩点须显式标注。
4. **定罪风险评估** —— 三档判断之一 + 依据（配套A）。
5. **辩护策略选择树** —— 主攻路径 + 选择理由（工具3，含三要素）。
6. **A/B 双方案** —— 主方案 + 备选方案及变数应对（配套B）。
7. **与后续文书的衔接说明** —— 本方案如何指导下游文书：明确"下一步应调用哪个文书 skill、该 skill 应主打哪几个辩点编号、需要补强哪些证据/论证"。

**报告头部元信息**：案件名称、案号（如有）、当事人类型（A类/B类）、诉讼阶段、上游材料来源（阅卷笔录/会见笔录文件名）、报告状态（讨论稿/已确认定稿）、生成日期。

## 四、执行流程

### 步骤 1：恢复上下文 & 读取上游材料
- 确认案件目录，读取 `CLAUDE.md` / `LOG.md` / 案件备忘录.md，恢复断点（如由 criminal-case-os 调度，继承其传入的案件路径与类型）。
- 读取 `criminal-case-review` 阅卷笔录（证据清单、证据编号、事实梳理、辩点线索）。
- 读取 `criminal-meeting` 会见笔录（如有）；无则记录"暂无会见笔录"。
- 确认 A/B 当事人类型与诉讼阶段。

### 步骤 2：提取辩点 + 填充矩阵 + 独立评估
- 从阅卷/会见材料中穷举所有辩点，编号 D1…Dn。
- 填充辩点矩阵（工具1），逐项回填证据编号。
- 逐辩点强度评级（工具2），写明依据。
- **独立立场**：标注弱辩点、不成立辩点、有反噬风险的辩点。
- 完成定罪风险评估（配套A）与策略选择树（工具3）草稿。

### 步骤 3：展示草稿 + 与律师讨论
- 向律师展示**矩阵 + 强度评估 + 定罪风险 + 策略选择树草稿**。
- 系统**主动给出独立建议**：推荐的主攻路径及理由，并明确指出哪些辩点不建议主攻。
- 与律师就主攻方向讨论。讨论中：
  - 律师有不同意见时，系统再次基于证据/法律评估，能让步则让步、不能让步则坚持并说明。
  - **讨论超过 3 轮，自动将讨论过程记录到「案件备忘录.md」**（争议点、系统意见、律师反馈、阶段结论）。

### 步骤 4：律师确认主路径 → 生成正式报告
- **必须等律师明确确认主攻路径**后，才生成定稿。
- 生成《辩护策略分析与路径选择报告》（7部分完整结构）。
- 文件名遵循 Windows 兼容规范（禁用冒号等非法字符，时间用短横线），保存到案件目录。
- 同步在「案件备忘录.md」追加"辩护方向决策"条目（决策结论 + 关键理由 + 确认时间）。
- 告知完整保存路径，并用 `code -r <文件路径>` 打开最终报告。

## 五、质量约束（定稿前逐条核验）

- [ ] **每个辩点都关联了**证据编号或法律条文，无空泛主张。
- [ ] 引用的法条**全部用元典 MCP（yuandian-law-search）验证**；未验证的标注 **【待元典验证】**，不得以未核实法条支撑结论。
- [ ] **无罪辩点与量刑辩点并存时，明确说明并列逻辑**（如"主张无罪为第一顺位，量刑情节为退而求其次的备位辩护，二者不冲突"），**不得自相矛盾**（不能一边主张完全无罪一边主动认罪求轻判而不作逻辑交代）。
- [ ] **系统独立立场已落地**：站不住的辩点已明确标注"强度弱/不建议主攻"或"不成立"，未迎合律师预设。
- [ ] 定罪风险评估给出了明确三档判断且有依据。
- [ ] 策略选择树每条路径写齐"触发条件/核心动作/对应后续文书"。
- [ ] A/B 双方案的备选方案覆盖了至少一种关键庭审变数。
- [ ] 第7部分明确指向了下游文书 skill 及应主打的辩点编号。
- [ ] 报告状态正确（未确认=讨论稿，已确认=定稿）。
- [ ] 重大讨论已记录到案件备忘录.md。

## 七、注意事项

- 法条引用必须经元典验证，未验证一律标注【待元典验证】，宁缺毋假。
- 当事人指称严格区分 A 类（受援人）/ B 类（委托人/当事人）。
- 全程不擅自定稿；主攻路径未经律师确认，报告状态恒为"讨论稿"。
- 系统独立立场是本 skill 的灵魂：真实评估优先于顺从，但最终决策权归律师。
