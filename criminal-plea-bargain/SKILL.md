# criminal-plea-bargain：认罪认罚协商策略意见书

## 一、技能定位

本 Skill 用于**审查起诉阶段**，帮助律师系统评估是否建议当事人选择认罪认罚从宽程序，并在决定认罪认罚的前提下，制定最优量刑建议协商策略，最终输出《认罪认罚协商策略意见书》供律师与当事人共同决策。

**触发场景**：
- 用户输入"认罪认罚""认罪认罚协商""量刑建议协商""要不要认罪认罚"等关键词
- 由 `criminal-case-os` 在审查起诉阶段调度
- 已有 `criminal-case-review` 输出的阅卷笔录（必要前提）

---

## 二、全局护栏（禁止违反）

| 护栏 | 具体要求 |
|------|---------|
| **辩护贯穿全程** | 认罪认罚分析不得放弃辩护视角，即便建议认罪，也要在量刑建议中最大化维权 |
| **系统独立立场** | 必须客观分析，不能只说"认罪认罚好"；不利方面必须清楚说明，不得隐瞒 |
| **当场解决问题** | 分析结论须具体可执行，不得给出"视情况而定"等无效建议 |
| **完整记录讨论** | 认罪认罚是重大不可逆决策，关键讨论轮次必须记录到案件备忘录 |
| **确认后才生成** | 正式意见书必须经律师确认且当事人知情后才能生成，禁止跳步 |

---

## 三、A/B 类型区分

**A类（法律援助）**：
- 受援人，须特别告知认罪认罚从宽制度的完整内容（权利、程序、后果）
- 须确认受援人系自愿且充分理解后再进入协商程序
- 须记录值班律师是否已提供法律帮助

**B类（委托）**：
- 委托人/当事人，由委托律师全程主导协商策略
- 重点在策略最优化，告知义务相对灵活

## 四、执行流程

```
Step 1  前置检查
        ├── 确认是否有 criminal-case-review 输出的阅卷笔录
        ├── 确认案件当前阶段（审查起诉/一审开庭前）
        └── 确认 A/B 类型

Step 2  输出【证据态势分析草稿】
        ├── 控方证据强度评估（强/中/弱）
        ├── 可用辩点汇总
        └── 定罪风险评估
        → 等待律师确认，有异议须修改

Step 3  输出【认罪认罚利弊分析表】
        ├── 认罪认罚方案利弊
        ├── 不认罪/无罪辩护方案利弃
        └── 系统独立给出倾向性建议（附理由，律师可推翻）
        → 与律师讨论，超过3轮关键分歧自动记录备忘录

Step 4  输出【量刑建议预测与协商策略】
        ├── 法定刑区间
        ├── 基准刑评估
        ├── 预期量刑区间
        └── 三层条款（底线/争取/交换筹码）
        → 律师确认协商边界

Step 5  当事人知情确认
        ├── 输出大白话版决策建议书
        ├── 告知具结书程序
        └── 记录当事人已知情确认

Step 6  生成正式《认罪认罚协商策略意见书》
        └── 保存路径：案件目录/intermediate/plea-bargain-strategy.md
```

## 五、输出结构：认罪认罚协商策略意见书

### 第一部分：案件证据态势分析

**控方证据强度评估**（强 / 中 / 弱）

从三性逐项分析：
- **合法性**：取证程序是否合规，有无非法证据排除空间
- **真实性**：书证、物证有无鉴真问题；证人证言有无矛盾
- **关联性**：证据指向构成要件的程度，是否有核心证据缺失

**辩方可用辩点汇总**

| 辩点类型 | 具体内容 | 证据支撑 | 实际效力评估 |
|---------|---------|---------|------------|
| 事实辩点 | | | |
| 证据辩点 | | | |
| 法律适用辩点 | | | |
| 程序辩点 | | | |

**定罪风险评估**：
- `大概率定罪`：核心证据充分，辩点改变结论可能性低
- `有争议`：存在实质辩点，结果存在不确定性
- `有可能无罪`：存在重大证据疑点或法律争议，值得无罪辩护

### 第二部分：认罪认罚利弊分析表

| 选择方案 | 有利方面 | 不利方面 |
|---------|---------|---------|
| **认罪认罚** | ① 量刑减让（通常10%–30%，特殊情况更多）<br>② 程序从简（速裁/简易程序）<br>③ 提前确定结果，减少不确定性<br>④ 可能缩短羁押期限<br>⑤ 检察官量刑建议法院通常采纳 | ① 放弃无罪辩护机会<br>② 形成有罪记录，影响后续（缓刑考验、职业资质等）<br>③ 量刑建议未必达预期，存在不认可风险<br>④ 压力下自愿性存疑（法律援助类须特别注意） |
| **不认罪 / 无罪辩护** | ① 保留无罪可能性<br>② 迫使控方严格举证，暴露证据漏洞<br>③ 维护当事人程序性权利<br>④ 有存疑利益归当事人原则保护 | ① 失去量刑减让<br>② 程序周期长，精神与经济压力大<br>③ 定罪后量刑可能更重<br>④ 羁押期限延长可能性高 |

**关于认罪认罚，你需要知道的事**

**一、认罪认罚是什么？**

认罪认罚不是"认输"，也不是放弃权利。它是法律给你的一个选择：如果你如实承认指控的罪行，并接受检察官提出的量刑建议，法院通常会判得比不认罪时轻一些。

**二、认罪认罚对你有什么好处？**

- 通常可以减少 10%–30% 的刑期
- 程序更快，不用等漫长的审判
- 结果更确定，你能提前知道大概判多久
- 如果符合条件，可能争取到缓刑

**三、认罪认罚有什么风险？**

- 一旦具结，就很难再主张"我是无辜的"
- 量刑建议是检察官给的，法院通常会采纳，但也可能不完全一致
- 会形成有罪的正式记录

**四、如果你选择不认罪呢？**

- 你可以要求检察官拿出所有证据，律师帮你逐条质疑
- 如果证据确实不够充分，你有机会无罪或者判得更轻
- 但如果最终还是被定罪，通常会判得比认罪认罚时更重

**五、这个决定只有你能做**

律师的职责是把情况分析清楚、把风险说明白，但最终是认罪认罚还是无罪辩护，**决定权在你**。

**请你考虑：**
- 你对指控的事实，是否有异议？
- 你现在的处境（羁押、家庭、工作），能否承受长期诉讼？
- 律师分析的证据情况，你是否认同？

**六、认罪认罚的程序你需要了解**

1. 律师与检察官协商确定量刑建议
2. 你签署《认罪认罚具结书》——**签字前必须仔细阅读**
3. 值班律师或辩护律师在场见证
4. 开庭时法院审查量刑建议是否合适
5. 法院判决

> ⚠️ **重要提醒**：具结书签署后不能随意撤回，请务必在充分理解后再决定。

## 六、质量约束

- **量刑预测必须有依据**：须引用法定刑区间+具体量刑情节，不得凭空给数字
- **法条引用须验证**：引用量刑规范性文件须通过元典MCP验证，未验证前标注【待元典验证】
- **策略建议须有事实支撑**：每条底线条款须说明依据，不得空洞
- **利弊分析须平衡**：认罪认罚与无罪辩护的分析必须均衡，不得只写有利方面
- **不得自相矛盾**：若建议认罪认罚，不得在量刑建议中同时为无罪辩护预留空间

## 升级版工具：协商情景模拟与推演

与检察官正式协商前，**预先推演检察官可能的三种反应**，针对每种反应准备好应对话术与让步顺序，避免临场被动、避免在压力下让出底线。本工具与第四部分的三层条款（底线 / 争取 / 交换筹码）联动使用。

### 一、三种情景预案

| 检察官反应 | 我方应对策略 | 让步底线 |
|-----------|------------|---------|
| **同意**（接受我方量刑建议） | 快速固定具结，确认底线条款逐项写入量刑建议书，核对措辞 | 守住底线条款不松动，不因检方"顺利"而追加让步 |
| **反对**（量刑建议高于我方预期） | 援引同类案例 + 量刑情节据理力争，请求降档；逐条回应检方加重理由 | 一旦触及底线条款，暂停协商，回头重新评估无罪 / 罪轻辩护路径 |
| **折中**（部分接受） | 在"争取条款"上主动让步，换取"底线条款"达成 | 用交换筹码（退赃 / 谅解 / 立功线索）置换核心诉求，绝不破底线条款 |

### 二、推演要点

1. **预设话术脚本**：每种情景预先写好开场、论证、回应、收口四段话术，确保团队多人协商时口径一致。
2. **固定让步顺序**：先让"争取条款" → 再动"交换筹码" → **绝不触碰"底线条款"**。让步必须换取对价，不做单方面退让。
3. **触线即停**：当协商触及底线条款且检方不让步时，立即暂停，不在现场承诺，回头评估转无罪 / 罪轻辩护的可行性。
4. **全程留痕**：协商全过程按谈话笔录模板记录（时间、在场人、检方意见、我方回应、阶段结论），关键轮次同步写入 `intermediate/case-memo.md`。

### 三、协商话术脚本模板（按情景填充）

