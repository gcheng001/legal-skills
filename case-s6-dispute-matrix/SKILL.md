# case-s6-dispute-matrix（S6）争点矩阵

## 工作定位

对比原告九步法与被告九步法主张，形成争点对撞，排序优先级（P0主战场/P1次战场/P2辅助），制定庭审精力分配建议。

**前置条件**：S5（主张检索）完成
**必须确认**：**是**（律师必审）
**北大法宝复验**：不需要
**双方共用**：S6争点矩阵双方共用同一份
**执行后**：调用Hook脚本更新状态

## 执行流程

### 第一步：读取双方文件

- 读取 `intermediate/原告九步法/` 下S1-S5所有文件
- 读取 `intermediate/被告九步法/` 下S1-S5所有文件

### 第二步：识别争议焦点

对比双方主张，识别争议焦点：
1. **事实争议**：双方对事实认定不一致
2. **法律争议**：双方对法律适用不一致
3. **程序争议**：管辖权、诉讼时效等

### 第三步：形成争点对撞

| 争点 | 原告主张 | 被告主张 | 关键证据 | 对撞结果 |
|------|----------|----------|----------|----------|
| 争点1 | ... | ... | ... | ... |
| 争点2 | ... | ... | ... | ... |

### 第四步：排序优先级

| 优先级 | 说明 | 庭审精力分配 |
|--------|------|--------------|
| P0 | 主战场，决定胜负 | 60% |
| P1 | 次战场，影响结果 | 30% |
| P2 | 辅助，锦上添花 | 10% |

### 第五步：生成S6文件

```markdown
# S6 争点矩阵

## 争议焦点概览
| 序号 | 争点 | 优先级 | 原告立场 | 被告立场 |
|------|------|--------|----------|----------|
| 1 | ... | P0 | ... | ... |

## 详细对撞分析
### 争点1：[争点内容]
- **原告主张**：[主张]
- **被告主张**：[主张]
- **关键证据**：[证据编号]
- **对撞结果**：[分析]

## 庭审精力分配建议
[建议内容]
```

保存到 `intermediate/原告九步法/S6-争点矩阵/`（双方共用）

### 第六步：律师必审

**红线**：AI不得自主落定，结构化状态保持 `pending_review`。

展示争点排序和策略，等用户确认。

### 第七步：调用Hook

```bash
~/.claude/skills/case-os/scripts/case-post-step.sh [案件路径] S6
```
