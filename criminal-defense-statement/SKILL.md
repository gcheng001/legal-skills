# criminal-defense-statement — 辩护词与庭审辩护

## 定位

一审阶段三合一交付模块，对应刑事案件OS交付物9：

| 序号 | 交付物 | 状态 |
|------|--------|------|
| 1 | 辩护词（完整版） | 律师说"可以出了"后生成 |
| 2 | 庭审辩论提纲（口袋版） | 随辩护词同步交付 |
| 3 | 最后陈述辅导材料 | 随辩护词同步交付 |

---

## 全局护栏（本模块严格度最高）

本模块适用刑事案件OS"五大原则"，且第5条在此处执行最严：

### 原则一：辩护贯穿全程
辩护词论点必须与前序文书（阅卷笔录、会见笔录、质证意见、不起诉意见）保持连贯一致。如有变化，必须说明变化原因。

### 原则二：系统独立立场
辩护词的核心辩点必须经过系统独立分析，不得照单全收律师意图。系统须对每个辩点主动评估：
- 该辩点的法律依据是否充分？
- 该辩点与证据是否相符？
- 该辩点与其他辩点是否自相矛盾？
- 若系统判断某辩点存在风险，**必须明确指出并说明理由**，不得因律师坚持而沉默。

### 原则三：当场解决问题
庭审辩论提纲须覆盖公诉人的可预期反驳路径，提前准备再反驳。

### 原则四：完整记录讨论
辩护词每次要点变更，均须记录：
- 变更内容是什么
- 变更原因（律师意见/新材料/法律依据更新）
- 对其他辩点的影响评估

### 原则五：确认后才生成（最严格执行）
**辩护词主文不得在律师明确说"可以出了"之前生成。**

修改阶段：若律师要求修改辩点，系统必须：
1. 追问：「这个调整的原因是什么？」
2. 分析：「此调整对以下辩点有影响：……，建议同步处理。」
3. 确认：律师确认后再修改。

## 执行流程

```
步骤1  材料核查
       ├─ 必须已读取：阅卷笔录、会见笔录
       ├─ 优先参考：质证意见（已有时）、不起诉意见（曾提交时）
       └─ 缺失必要材料时，明确告知律师并暂停

步骤2  系统出具辩护方案草案（独立立场）
       ├─ 识别可用辩点类型（四类，见下）
       ├─ 逐条说明法律依据和证据支撑
       ├─ 标注存疑或风险辩点
       └─ 输出：「辩护方案草案」，格式见附录A

步骤3  与律师讨论
       ├─ 律师同意 → 进入步骤4
       ├─ 律师调整辩点 → 追问原因 → 影响分析 → 确认 → 更新草案
       └─ 记录每次变更（附录B：变更日志）

步骤4  律师明确说"可以出了"
       └─ 触发词：「可以出了」/「出文书」/「生成辩护词」（明确指令）

步骤5  同步生成三个交付物
       ├─ 交付物1：辩护词（完整版）
       ├─ 交付物2：庭审辩论提纲（口袋版）
       └─ 交付物3：最后陈述辅导材料

步骤6  文件存档
       └─ 保存路径：案件目录/intermediate/09_辩护词_[YYYYMMDD].md
```

### 辩点结构（四类，按实际选用）

#### 辩点类型一：事实不清、证据不足（无罪辩护）

适用情形：证据体系存在实质缺口，无法排除合理怀疑。

论证框架：
```
一、控方指控事实XXX缺乏充分证据支撑

（一）直接证据层面：……（逐项分析，引证据编号和页码）
（二）间接证据层面：……（论证证据链断裂点）
（三）证据间矛盾：……（指出具体矛盾）
（四）综合评价：上述证据不能形成完整锁链，无法排除……的合理怀疑。

依据：《刑事诉讼法》第55条（证据裁判原则）；《关于适用<中华人民共和国刑事诉讼法>的解释》第XX条。
```

禁止事项：
- 不得笼统说"证据不足"，必须指出具体断裂点
- 不得无根据否定鉴定意见，须援引反驳依据

#### 辩点类型二：定性辩护（此罪与彼罪）

适用情形：行为事实基本认可，但指控罪名不当。

论证框架：
```
二、被告人的行为不符合XXX罪的构成要件

（一）客观要件：指控行为缺少……要素（具体说明）
（二）主观要件：被告人主观上……（结合会见陈述和客观证据）
（三）与相邻罪名的区别：本案与XXX罪的区别在于……
（四）应认定为：若有事实，最多构成XXX，理由……

依据：《刑法》第XX条；相关司法解释第XX条；参考案例……
```

注意：同时提出无罪和定性辩护时，必须在辩护词开头说明并列策略的逻辑，避免给合议庭混乱印象。

#### 辩点类型三：量刑辩护（罪轻情节）

适用情形：认罪或不争定性，争取从轻量刑。

各项情节论证要点：

**自首：**
```
被告人XXX于XX年XX月XX日……（到案经过），符合《刑法》第67条第1款规定的自首条件。
具体体现：（一）自动投案：……；（二）如实供述：……
建议依法从轻或减轻处罚。
```

**坦白：**
```
被告人到案后如实供述主要犯罪事实，符合《刑法》第67条第3款规定的坦白条件。
（与自首互斥，若主张自首则不主张坦白）
```

**立功：**
```
被告人在羁押期间……（具体立功事实），经侦查机关核实，符合《刑法》第68条立功认定条件。
```

**退赃退赔：**
```
被告人（或其家属）已于XX年XX月XX日退赔被害人人民币XXX元（见谅解书/转账记录），表明悔过诚意，应酌情从轻处罚。
```

**谅解：**
```
被害人XXX出具书面谅解书（见附件），谅解书载明……，依据《刑法》及相关司法解释，谅解情节应当从轻处罚。
```

**认罪认罚：**
```
被告人自愿认罪认罚，签署具结书时程序合法、意思表示真实自愿（辩护人在场见证），应依法从宽处理。
```

**从犯地位：**
```
在共同犯罪中，被告人系从犯，具体表现：……（起意者非被告人/作用次要/参与程度有限），依据《刑法》第27条应当从轻、减轻或者免除处罚。
```

#### 辩点类型四：程序辩护（程序违法）

适用情形：存在非法证据或重大程序瑕疵。

论证框架：
```
四、关于程序问题

（一）非法证据排除申请：
    本辩护人于庭前已就……证据提出排除申请（见排非申请书）。
    该证据系通过……方式取得，违反……规定，应予排除。
    若该证据被排除，控方证据体系……

（二）程序瑕疵对证明力的影响：
    侦查机关在……环节存在……瑕疵（具体说明），
    依据……规定，该证据证明力存疑，不应单独作为定案依据。
```

### 辩点并列策略说明（需要时）

当同时采用无罪辩护和量刑辩护时，辩护词开头须加以下说明：
```
辩护人注意到，本辩护意见同时提出无罪辩护和量刑辩护意见。提出量刑辩护意见，并非辩护人认为被告人有罪，而是基于对司法实践的审慎考量，在法庭不采纳无罪辩护意见的情形下，充分保障被告人的合法权益。
```

## 交付物3：最后陈述辅导材料

### 三版陈述模板

**版本一（认罪/认罪认罚时）：**
```
审判长、审判员，我对自己做的事情非常后悔。我想对被害人说一声对不起。我以后一定好好改造，不再做违法的事。恳请法庭给我一个改过自新的机会。谢谢法庭。
```

**版本二（无罪辩护时）：**
```
审判长、审判员，我没有做过起诉书指控的事。我相信法律是公正的，会还我清白。无论结果如何，我都会尊重法庭的判决。谢谢。
```

**版本三（定性辩护/轻罪辩护时）：**
```
审判长、审判员，我做错了事，我认。但我做的事和起诉书说的不一样。我诚恳地请求法庭实事求是地认定事实，对我做出公正的判决。谢谢。
```

### 庭前辅导要点

**表达方式：**
- 语速：比正常说话慢一点，稳定清晰
- 眼神：始终面向法官席，不看公诉人或旁听席
- 态度：诚恳、克制、不卑不亢
- 声量：清晰可闻即可，不需要刻意高声

**禁止事项：**
- 最后陈述时不翻供（与庭审陈述矛盾）
- 不攻击、谩骂公诉人或证人
- 不长篇陈述案情（最后陈述不是再次质证）
- 不泄露辩护策略细节

**会见时辅导流程：**
1. 告知被告人三个版本的适用情形，共同选择
2. 让被告人复述1-2遍，纠正语气和表述
3. 提示在法庭上若忘词，沉默3秒再说，不要慌乱

## 升级版工具：模块化辩护词 A/B 方案

### 要解决的问题

庭审现场存在变数（证据采信情况、证人当庭表现、公诉人量刑建议调整等），单一定稿的辩护词难以灵活应对。本工具将辩护词拆分为可独立增删、快速重组的模块，预置应变版本，使辩护人能在庭上快速抽换、插入对应模块。

### 红线（与本模块第五原则一致，不得突破）

- **A/B 方案的模块切换属于辩护词内容，同样适用"律师明确说'可以出了'之后才生成"的最高红线。** 在律师确认前，仅可输出模块结构规划与切换决策表，不得生成任何模块的辩护词正文。
- 修改、抽换、插入任一模块，均按第五原则执行：追问原因 → 影响分析 → 律师确认 → 记录变更日志（附录B）。
- **无罪模块与量刑模块并存时不得自相矛盾**：量刑模块须沿用"辩点并列策略说明"的退让逻辑（见前文），表述为"在法庭不采纳无罪意见的情形下"，不得出现默认有罪的措辞。

### 四大模块划分

辩护词正文按以下四大模块独立成段，每段标注模块标签，便于现场抽换：

| 模块标签 | 模块名称 | 对应辩点类型 | 可独立抽换 |
|----------|----------|--------------|------------|
| 【M1·事实】 | 事实认定模块 | 事实层面陈述与争点 | 是 |
| 【M2·证据】 | 证据分析模块 | 辩点类型一/四（证据缺口、非法证据） | 是 |
| 【M3·法律】 | 法律适用模块 | 辩点类型二（定性、此罪彼罪） | 是 |
| 【M4·量刑】 | 量刑情节模块 | 辩点类型三（罪轻情节） | 是 |

每个模块自成闭环：开头点明该模块主张，中间逐条论证（引证据编号、法条），结尾一句话收束，确保抽出或插入时不影响其余模块的连贯性。

### A/B 双方案

> 注意：此处 A/B 是**辩护词文本层面的两个版本**，区别于 criminal-defense-strategy 的策略路径主备方案（那是策略路径层面）。本工具只管文本模块的现场重组。

**A方案（基准版）：** 基于最可能出现的庭审情形撰写的完整辩护词，四大模块齐备，为庭审默认宣读版本。

**B方案（应变版）：** 针对庭审可能出现的变数，预先准备的替换模块（只准备会变动的模块，不重复撰写全文）。常见触发与对应替换：

| 庭审变数 | 触发条件 | 启用的 B 模块 | 替换/插入对象 |
|----------|----------|---------------|----------------|
| 关键证据被法庭采信 | 排非申请未获支持、争议证据被确认 | "退守量刑辩护"模块（强化 M4） | 替换/补强 A方案 M2、M4 |
| 证人当庭翻证 | 原供述证人当庭推翻证言 | "强化无罪/存疑"模块（强化 M2） | 替换 A方案 M2 |
| 公诉人变更量刑建议 | 当庭加重/减轻量刑建议 | 对应量刑回应模块（调整 M4） | 替换 A方案 M4 |

B方案模块编号建议：`M2-B`、`M4-B` 等，与 A 方案同模块号对应，便于现场一一替换。

### 重组规则

1. 四大模块各自独立成段并标注模块标签，互不嵌套，保证可单独抽换。
2. 庭审中根据实际情况，从 A 方案快速切换/插入 B 方案对应模块；切换以"整模块替换"为单位，不做段内拼接，避免逻辑断裂。
3. 切换后须即时检查无罪模块（M2）与量刑模块（M4）是否并存——若并存，确认量刑模块仍采用退让表述，无自相矛盾。
4. **每一次切换决策（触发条件 + 启用模块 + 决策人）写入"庭审辩论速查手册（口袋版）"**，与交付物2联动，使辩护人庭上一眼可查"什么情况下换哪个模块"。

### 与口袋版提纲的联动：庭审辩论速查手册

在交付物2（口袋版提纲）中增设"模块切换速查"区块，供庭上即时决策：

```
├─────────────────────────────────────────┤
│ 【模块切换速查 A→B】                       │
│ 若[关键证据被采信] → 换 M4-B（退守量刑）   │
│ 若[证人当庭翻证]   → 换 M2-B（强化存疑）   │
│ 若[公诉人改量刑建议]→ 换 M4-B（量刑回应）  │
│ 切换后必查：M2无罪 与 M4量刑 不冲突        │
└─────────────────────────────────────────┘
```

### 执行衔接

本工具嵌入主执行流程，不另起独立流程：

- **步骤2（出草案）时**：在辩护方案草案中即标注四大模块划分，并预判需准备哪些 B 模块（仅规划，不出正文）。
- **步骤3（与律师讨论）时**：与律师确认 A 方案模块构成、需预置的 B 模块及其触发条件，记录入变更日志。
- **步骤4（律师说"可以出了"）后**：方可生成 A 方案完整辩护词 + B 方案各替换模块正文 + 口袋版"模块切换速查"。**未确认前，B 模块正文一律不得生成。**

### 存档补充

A/B 模块化辩护词在原存档基础上增加应变模块文件：

```
{案件目录}/intermediate/
├── 09_辩护词_A基准版_{YYYYMMDD}.md       # A方案完整辩护词
├── 09_辩护词_B应变模块_{YYYYMMDD}.md     # B方案各替换模块（含触发条件）
└── 09_庭审辩论提纲_{YYYYMMDD}.md          # 含"模块切换速查"区块
```
