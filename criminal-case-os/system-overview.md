# 刑事案件OS体系总览

## 概述

刑事案件OS（criminal-case-os）是刑事案件办理的完整操作系统，包含总控调度器和多个子技能模块，以**刑事诉讼真实阶段**为主线，覆盖从侦查（拘留→批捕）、审查起诉到一审的全流程辩护工作。

本系统采用**四阶段双层结构**，全部模块已就绪、无任何待建项；**二审阶段已明确排除，不在本系统范围内**。

**双轨体系**是本系统的核心维度：

- **对内轨道（办案文书，9个刑事skill）**：服务于律师办案，输出阅卷笔录、辩护方案、质证发问、辩护词等专业文书，要求专业、犀利、底牌完整。
- **对外轨道（客户可视化，criminal-client-comm）**：服务于客户沟通，输出服务计划书、工作通报、可视化时间轴、庭审简报等。最高红线是**脱敏不泄底牌**——可以让客户感知工作进展与价值，但绝不暴露辩护策略与诉讼底牌。

全系统共 **10个刑事子skill + 2个通用准备skill + 5份共享资源**。

## 体系架构

```
criminal-case-os（总控）
    ↓
准备组
  ├── 案件初始化（case-git-init）
  └── 基础信息提取（case-info-extract）

一、侦查阶段（拘留→批捕，无卷）
  └── criminal-investigation（无卷会见提纲与辅导、取保/不批捕意见书、
        羁押必要性审查；含逮捕必要性评估表、首次会见分析报告）

二、审查起诉阶段（检察院）
  ├── criminal-case-review（阅卷笔录：A类12部分 / B类证据分析报告式8部分）
  ├── criminal-defense-strategy（辩护方案：辩点矩阵+强度评估+策略选择树）
  ├── criminal-non-prosecution（不起诉意见：法定/存疑/酌定；含类案检索、提交计划）
  └── criminal-plea-bargain（认罪认罚协商：利弊分析+情景推演+决策辅助）

三、一审阶段（法院）
  ├── criminal-meeting（阅卷后会见发问清单）
  ├── criminal-trial-examination（庭前会议+排非、庭审发问+质证；
        含质证指令卡、三板斧、总分总质证法）
  ├── criminal-defense-statement（辩护词+辩论口袋版+最后陈述；含模块化A/B方案）
  └── criminal-case-visualization（全套8张图表，全程可用）

贯穿全程·对外轨道
  └── criminal-client-comm（客户沟通与服务可视化：服务计划书/工作通报/
        可视化时间轴/决策辅助/庭审简报；最高红线=脱敏不泄底牌）

共享资源（references/）
  ├── 庭审当日时间线.md
  ├── 非法证据排除速查.md
  ├── 刑事文书格式规范.md
  ├── 刑事法律依据索引.md
  └── 全流程风险速查.md
```

- 统一入口和调度器

- 判断案件所处诉讼阶段，按阶段调度对内/对外两轨子技能
- 维护共享资源（references/）供各子skill调用

**触发词**：
- 刑事OS
- 刑事案件OS
- criminal-case-os

**核心特点**：
- 五大核心原则（辩护贯穿全程、系统独立立场、当场解决问题、完整记录讨论、确认后才生成）
- 四阶段双层结构（准备组 + 侦查 + 审查起诉 + 一审），二审已排除
- 双轨体系（对内办案 9 skill + 对外客户可视化 criminal-client-comm）
- 双端独立运行与断点接续
- 类型选择（A类法律援助 / B类委托）

### 3. criminal-case-review（刑事阅卷）

**文件**：`~/.claude/skills/criminal-case-review/SKILL.md`

**功能**：
- 制作阅卷笔录
- 证据归属核验
- 原件优先纠偏
- 分步撰写工作流

**触发词**：
- 刑事阅卷
- 刑事阅卷笔录

**子目录结构**：
```
criminal-case-review/
├── SKILL.md                       # 通用规范
├── agents/                        # Agent配置
│   ├── openai.yaml               # 法援律师（A类）
│   └── entrust.yaml              # 委托律师（B类）
└── references/                    # 参考资料
    ├── type-b-differences.md     # B类详细差异说明
    ├── type-b-workflow.md        # B类专门工作流程
    └── type-b-report-template.md # B类证据分析报告式模板（8部分）
```

**核心特点**：
- 支持A类（法律援助）和B类（委托）两种类型，结构不同：
  - **A类：标准12部分结构**（完整阅卷笔录）
  - **B类：证据分析报告式8部分结构**（封面 → 案件信息摘要 → 全案证据分类清单 → 逐证三性分析表 → 证据链整体评估 → 程序违法审查记录 → 辩护思路 → 落款）
- **质量护栏全部移植**：B类虽改用8部分结构，但12部分原有的质量护栏（页码真实性、原件优先、讯问八要素等）全部移植到B类，质量标准不降低
- 逐部分撰写、逐部分确认
- 原件优先纠偏规则
- 供述归属校核规则

### 4. criminal-defense-strategy（辩护方案）

**文件**：`~/.claude/skills/criminal-defense-strategy/SKILL.md`

**功能**：
- 基于阅卷成果制定系统化辩护方案
- 填补"阅卷与辩护词之间"的辩护方案决策环节
- 辩点梳理、强度评估、策略路线选择

**触发词**：
- 辩护方案
- 辩护策略
- 辩点分析

**核心特点**：
- **辩点矩阵**：系统化罗列无罪/罪轻/程序/量刑各类辩点
- **强度评估**：对每个辩点的证据支撑度、采纳概率进行评级
- **策略选择树**：在不同辩护路线（无罪辩护 / 罪轻辩护 / 量刑辩护 / 程序辩护）之间提供结构化决策辅助
- 系统独立立场——客观评估各路线利弊，不盲目迎合

### 5. criminal-non-prosecution（不起诉意见）

**文件**：`~/.claude/skills/criminal-non-prosecution/SKILL.md`

**功能**：
- 审查起诉阶段撰写不起诉意见书
- 区分法定不起诉 / 存疑不起诉 / 酌定（相对）不起诉三类情形

**触发词**：
- 不起诉
- 不起诉意见

- **三类型区分**：法定不起诉、存疑不起诉、酌定不起诉，依不同法律事由分别论证
- 内置**类案检索**流程，以同类不起诉案例增强说服力
- 含**提交计划**，明确向检察院提交的时机与方式
- 与辩护方案、阅卷成果联动

## 类型差异（A类 vs B类）

| 项目 | A类：法律援助类 | B类：委托类 |
|------|------------------|--------------|
| 称谓 | 受援人 | 委托人 / 当事人 |
| 指派来源 | 法律援助中心指派 | 当事人或家属委托 |
| 文书抬头 | 受援人：XXX | 委托人：XXX |
| 律师身份 | 法律援助律师 | 委托辩护人 |
| 费用说明 | 法律援助不收取费用 | 按委托合同约定 |
| 归档规范 | 法律援助案卷归档规范 | 律所内部归档规范 |
| **阅卷笔录结构** | **标准12部分** | **证据分析报告式8部分** |

**阅卷结构差异说明**：

- **A类（12部分）**：完整阅卷笔录标准结构。
- **B类（8部分证据分析报告式）**：封面 → 案件信息摘要 → 全案证据分类清单 → 逐证三性分析表 → 证据链整体评估 → 程序违法审查记录 → 辩护思路 → 落款。模板见 `criminal-case-review/references/type-b-report-template.md`。
- **质量标准一致**：B类虽结构精简，但12部分的质量护栏（页码真实性、原件优先、讯问八要素等）全部移植到B类，质量不打折。

**详细说明**：
- A类详细规范见阅卷笔录通用规则
- B类详细规范见 `criminal-case-review/references/type-b-differences.md`、`type-b-workflow.md` 和 `type-b-report-template.md`

## 标准工作流程

### 准备组（案件受理与准备）

1. **案件初始化**
   - 创建案件目录和Git仓库
   - 调用 `case-git-init` skill

2. **案件基础信息提取**
   - 提取当事人、案号、罪名等信息
   - 调用 `case-info-extract` skill
   - **必须律师确认**

### 一、侦查阶段（拘留→批捕，无卷）

3. **侦查阶段辩护**
   - 无卷会见提纲与辅导、取保/不批捕意见书、羁押必要性审查
   - 调用 `criminal-investigation` skill
   - 含逮捕必要性评估表、首次会见分析报告

### 二、审查起诉阶段（检察院）

4. **刑事阅卷**
   - 制作阅卷笔录（A类12部分 / B类8部分报告式）
   - 调用 `criminal-case-review` skill
   - **逐部分确认**

5. **辩护方案**
   - 基于阅卷成果制定辩护方案（辩点矩阵+强度评估+策略选择树）
   - 调用 `criminal-defense-strategy` skill

6. **不起诉意见**（视情形）
   - 撰写法定/存疑/酌定不起诉意见书
   - 调用 `criminal-non-prosecution` skill

7. **认罪认罚协商**（视情形）
   - 利弊分析、情景推演、决策辅助
   - 调用 `criminal-plea-bargain` skill

### 三、一审阶段（法院）

8. **刑事会见**
   - 基于阅卷笔录生成发问清单
   - 调用 `criminal-meeting` skill
   - **必须律师确认**

9. **庭审质证与发问**
   - 庭前会议+排非、庭审发问+质证
   - 调用 `criminal-trial-examination` skill

10. **辩护词**
    - 撰写辩护词+辩论口袋版+最后陈述
    - 调用 `criminal-defense-statement` skill
    - **必须律师确认**

11. **刑事可视化**（全程可用）
    - 生成全套8张图表
    - 调用 `criminal-case-visualization` skill
    - 生成后律师审阅

### 贯穿全程·对外轨道

- **客户沟通与服务可视化**
  - 服务计划书/工作通报/可视化时间轴/决策辅助/庭审简报
  - 调用 `criminal-client-comm` skill
  - **脱敏不泄底牌**

## 全局规则

### 五大核心原则

1. **辩护贯穿全程**：辩护不是单独步骤，是贯穿全流程的工作方式
2. **系统独立立场**：系统必须独立分析利弊，不盲目顺从
3. **当场解决问题**：每步发现的问题必须在该步解决
4. **完整记录讨论**：讨论过程必须记录到案件备忘录
5. **确认后才生成**：辩护词等重要文书必须律师确认后生成

### 全局红线

1. **文件操作铁律**：未经明确允许，绝对禁止删除或替换任何文件
2. **文件夹命名铁律**：禁止使用emoji和特殊字符
3. **确认机制红线**：案件基础信息、阅卷笔录、辩护词必须律师确认
4. **Git管理铁律**：每个步骤完成后自动git提交
5. **对外脱敏红线**：对外轨道（criminal-client-comm）一切输出必须脱敏，绝不泄露辩护策略与诉讼底牌
