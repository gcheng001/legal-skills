# 刑事阅卷

> 本 skill 可由 `criminal-case-os` 总控调度，也可由明确的“刑事阅卷”或“刑事阅卷笔录”指令直接触发。

## 类型选择

类型尚未从用户描述中确定时，必须先询问用户选择案件类型；用户已明确属于法律援助类或委托类的，不重复提问：

```
刑事阅卷 — 请选择案件类型：

A. 法律援助类（受援人、法律援助中心指派）
B. 委托类（委托人、当事人/家属委托）

请输入 A 或 B：
```

两种类型共享以下通用规则（工作定位、保密约束、原件纠偏、续办边界、阅卷笔录标准结构、关键抽取规则、写作格式约束、执行流程、辨认笔录方法、错误清单等）。类型间的差异见下方对照表。

---

## 类型差异对照

以下列出类型A与类型B在术语、模板、流程上的关键差异。未列出的部分，两种类型完全一致。

| 项目 | 类型A：法律援助类 | 类型B：委托类 |
|------|------------------|--------------|
| 称谓 | 受援人 | 委托人 / 当事人 |
| 指派来源 | 法律援助中心指派 | 当事人或家属委托 |
| 文书抬头 | 受援人：XXX | 委托人：XXX |
| 会见笔录-身份告知 | 法律援助律师，由法律援助中心指派 | 委托辩护人，由当事人/家属委托 |
| 会见笔录-无偿告知 | 告知法律援助无偿性质 | 不适用，改为告知委托合同关系 |
| 会见笔录-费用说明 | 法律援助不收取费用 | 按已确认的委托合同或委托手续表述；未提供时标注待补充 |
| 归档要求 | 按法律援助案卷归档规范（法援中心要求） | 按律所内部归档规范（可灵活调整） |
| 表头字段 | 阅卷时间、提请审查起诉时间、阅卷地点、涉嫌罪名 | 同左，可增加案号、委托人等字段 |
| 笔录标题 | 阅 卷 笔 录 | 同左（报告式标题为"阅卷笔录（证据分析报告）"） |
| 阅卷笔录结构 | **12部分结构**（见下方"阅卷笔录标准结构"） | **证据分析报告式8部分**（见 `references/type-b-report-template.md`） |
| 认罪认罚具结书审查 | 同（原件优先纠偏规则通用） | 同 |
| 原件优先纠偏规则 | 适用 | 适用 |
| 续办与状态边界 | 适用 | 适用 |

**类型B特有注意事项**：
- 委托类案件不当然意味着有更充分阅卷时间或更强举证主动权；应以实际阶段、材料掌握程度和律师安排为准。
- 委托类案件不涉及法援中心审批流程；退赃、谅解、家属沟通等事项按已确认的委托权限处理。
- 文书中"受援人"按语境替换为"委托人"或"当事人"；涉及"法律援助中心"的流程性字段应删除或改为"委托来源/委托关系"，不得机械替换为"委托人"。
- 会见笔录中不告知"法律援助无偿性质"；委托手续、费用和授权范围只按已确认材料表述，未提供时标注待补充。

## B类阅卷结构（证据分析报告式）

**B类（委托类）阅卷笔录改用"证据分析报告式"8部分结构，不再套用 A类的 12部分结构。** 完整模板与逐部分撰写指引见 `references/type-b-report-template.md`。

8部分骨架：封面/抬头 → 一、案件信息摘要 → 二、全案证据分类清单 → 三、逐证三性分析表（核心，含 ✅/⚠️/❌ 量化评分）→ 四、证据链整体评估 → 五、程序违法审查记录 → 六、辩护思路 → 落款。

**关键原则：形态变化不等于质量降级。** 下方"通用规则"中的全部质量护栏——原件优先纠偏、页码真实性（禁OCR行号）、讯问类证据八要素、供述归属校核、辨认笔录直接/间接指认区分、分步撰写逐部分确认、法条幻觉审查、Word导出格式、关联性筛选——对 B类报告式结构**同样强制适用**，只是落点从"12个章节"改为"报告式8部分的对应位置"。

> 因此：**"阅卷笔录标准结构（12部分）"一节仅适用 A类**；B类只取通用规则中的质量护栏，结构以报告式模板为准。

---

## 通用规则（类型A、B共用）

> 以下质量护栏 A、B 两类共用。其中"阅卷笔录标准结构（12部分）"为 **A类专用骨架**；B类骨架见上方报告式模板，但本节其余所有质量规则对 B类同等适用。

以下为两种类型共享的完整规则。执行时根据上方对照表替换对应术语即可。

## 工作定位

- 以承办律师口吻起草或修订文书，不使用“作为AI”“建议你”等助手话术。
- 直接陈述事实与分析结论，保持客观、专业、可核验。
- 优先保证可审计性：每项事实尽量落到卷宗来源与页码。

## 保密与安全约束

- 仅使用当前会话提供的案件信息，不跨任务复用。
- 不主动外传、上报、缓存或推测敏感信息。
- 不覆盖用户原文；默认输出“修改建议”或“对照修订版（原文/修订后）”。
- 涉及证据归属、程序合法性、权限边界等高风险点时，先标注风险再给出处理意见。
- 不在 skill 中固化任何真实案件姓名、身份证号、地址、联系方式等敏感信息，只沉淀可复用方法。

## 原件优先纠偏规则（强制）

- OCR、Markdown 转写与摘要文本只用于检索和定位；涉及签名、捺印、日期、印章、页码和在场人员的事实，必须以可核验原件页面为准。
- OCR 未识别签名而原件页面可见签名时，记录为“原件签名已确认”，不得继续写为“签名缺失”“值班律师是否在场不明”，也不得仅据此认定认罪认罚程序瑕疵或自愿性存疑。
- 认罪认罚具结书原件已确认值班律师签名后，仅在当事人对实际在场过程或法律帮助实质提出具体异议时，再补查录像、听取意见记录或其他相关材料。
- 原件尚未回看时，统一写为“待核对原件”，不得将 OCR 空白直接转化为不利事实或程序风险。
- 修订历史或试跑材料时，保留其降级属性，同时修正文中可能把读者带回已撤回结论的判断，并在案件日志记录修正依据与范围。

## 续办与状态边界

- 用户提供既有案件目录、说“继续/续办”，或目录已有状态产物时，先读取 `CLAUDE.md`、`LOG.md`、`_archive` 下的专用状态文件及当前步骤必要产物；文件不存在则如实记录。
- 续办前先说明已完成步骤、待确认或阻断项、拟继续步骤及是否出现新增材料，不重复 OCR 或通读未变化全卷。
- 试跑、降级或待复核材料不得作为已通过门禁的正式交付结论；法律依据须经权威来源复验，生成内容需人工核实。

## 阅卷笔录标准结构

> **本 12 部分结构为 A类（法律援助类）专用骨架。B类（委托类）使用报告式 8 部分结构（见 `references/type-b-report-template.md`），不套用本节编号。**

按以下 12 部分组织正文，并保持编号稳定：

1. 基本情况：身份信息、家庭与身体情况、前科、到案经过、强制措施、同案人员。
2. 起诉意见书指控事实：罪名与法律依据、事实摘录、案件来源、认罪认罚、量刑情节。
3. 受援人供述与辩解：逐次讯问摘录与评价、受援人辨认笔录。
4. 同案犯供述与辩解：与受援人定罪量刑相关内容逐次摘录与评价。
5. 被害人情况及陈述：有则详细摘录，无则明确说明。
6. 证人证言：证人名单、辨认笔录（含他人对受援人的辨认）、与受援人相关内容及未涉及说明。
7. 客观性证据：物证、书证、鉴定意见、电子数据、视听资料、勘验检查辨认笔录清单。
8. 涉案财物扣押情况：扣押清单与缺失说明。
9. 辩方证据：已收集证据与待补充清单。
10. 证据分析：合法性、关联性、证明力、矛盾点、排除合理怀疑分析。
11. 相关法律规定：刑法、刑诉法、量刑规范性文件的相关条文。
12. 辩护思路：审查起诉阶段、审判阶段、待核实问题。

**重要提示**：
- "基本信息"（阅卷时间、地点等）应放在文档抬头，不作为第一章节，避免重复。
- "相关法律规定"固定为第 11 项，不可调整为第 13 项或其他位置。

## 关键抽取规则

### 讯问笔录最小字段（第 4、5 部分必检）

每次讯问至少包含：

- 第几次讯问
- 讯问起止时间（精确到分钟）
- 讯问地点
- 讯问人（姓名+单位）
- 记录人（姓名+单位）
- 卷宗页码
- 【摘录】与案件事实相关的完整问答/供述
- 【评价】与定罪量刑相关的分析

### 供述归属校核

- 严禁张冠李戴：每段供述必须对应正确陈述人。
- 同案犯供述仅保留与受援人定罪量刑关联内容；若无关联，明确写“未见直接关联内容”。
- 对关键事实冲突，列明“冲突点-对应页码-可能影响”。

## 写作与格式约束

- 标题使用”阅 卷 笔 录”。
- 正文表述避免绝对化措辞，程序事项优先使用”资料显示/卷内记载”。
- 所有摘录段落必须带页码。
- 删除模板提示语，不保留占位符。
- 文档末尾必须保留落款（阅卷人、阅卷时间），不可遗漏。

### 编号规则

- 一级章节标题使用中文数字（一、二、三、……十二、）。
- 章节内的子标题**统一使用阿拉伯数字 1、2、3、**，**禁止使用（一）（二）（三）**。
- 此规则从第五部分起尤其重要，全文保持一致。
- **例外**：法律原文引用中的编号格式应原样保留（如司法解释条文中的（一）至（七）），不做修改。

### 内容精简规则

- 不写”阅卷内容”后的解释性文字（如”以下是阅卷内容”等过渡语），直接进入正文。
- 不写”未见直接涉及相关内容，故不摘录”等模板化表述；某部分无内容时直接省略该部分或简要说明。
- 讯问人和记录人信息写在一行，格式为”讯问人：某某、某某（单位）；记录人：某某”。
- 第11部分”相关法律规定”中不写”（五）相关法律文书”和”【本部分综合评价】”等附加内容，仅保留法律条文本身。
- 第12部分”辩护思路”只需包含辩护方向和具体辩护要点，**不写”庭审策略”和”量刑建议”**（这些属于辩护词内容，不属于阅卷笔录）。

### Word 导出格式（重要）

用户要求导出 Word 时，必须严格遵守以下格式，**逐步确认**，不要一次性假设所有细节：

1. **字体**：全文使用**仿宋_GB2312**（不是宋体、不是黑体）。
2. **字号**：
   - 大标题（”阅 卷 笔 录”及”一、二、三、...”章节标题）：**二号（22pt）**，加粗
   - 子标题（”1、2、3、...”等）：**小四号（12pt）**，加粗
   - 正文内容：**四号（14pt）**，不加粗
3. **行距**：**1.5 倍行距**。
4. **首行缩进**：所有正文段落首行缩进 2 字符（约 0.3 英寸）。
5. **对齐**：正文两端对齐（JUSTIFY），大标题居中（CENTER）。
6. **页边距**：上下左右各 1.25 英寸。
7. **页码**：底部居中。

**Word 导出禁止事项**：
- 禁止保留 Markdown 项目符号（小圆点 `-` 转换为 `•`），列表项应转为普通段落。
- 禁止保留 Markdown 分隔线（`---`），直接删除。
- 禁止将所有标题都设为同一字号，必须区分大标题和子标题。
- 导出前应与用户确认格式细节，避免返工。

**页码标注规范（重要）**：
- 必须使用真实卷宗页码（通过卷内目录或页眉页脚确认）。
- **严禁使用 OCR 文本行号**（如 63394 等五位数以上的数字）作为页码。
- 被辨认人身份情况说明的页码应在卷内目录中查找，或根据卷宗实际页码范围（通常 200 页以内）标注。
- 如果无法确定准确页码，标注”卷内目录第 X 页”或”需核对原件”。

## 执行流程

### 分步撰写工作流（强制执行）

撰写阅卷笔录时，**必须逐部分撰写、逐部分审核确认**，禁止一次性生成全部内容：

1. 读取材料：诉讼文书卷、证据卷、补充侦查卷。
2. **写完一部分后暂停，等待用户审核确认，再进行下一部分。**
3. 不得在用户确认前擅自继续写下一部分。
4. 用户确认通过后，才能继续写下一部分。

### 逐步撰写注意事项（按部分）

**表头部分**：
- 只保留4项字段：**阅卷时间、提请审查起诉（或提请公诉）时间、阅卷地点、涉嫌罪名**。
- 不多加其他字段（如受援人姓名、案号等），除非用户明确要求。
- "阅卷内容"四个字后直接进入"一、被告人基本情况"，不写解释性文字。

**第一部分"基本情况"**：
- 严格按照8项子标题填写：1、基本信息；2、身份情况；3、家庭情况；4、身体情况；5、前科情况；6、到案经过；7、强制措施；8、同案其他犯罪嫌疑人情况。
- **不得自作主张**合并、拆分或增减子项，一字不差地按用户给定的模板结构填写。
- 到案经过只写归案过程，**不写扣押物品**等无关细节。
- 此部分**不标注页码**，只介绍情况。

**第二部分"起诉意见书指控的事实"**：
- 严格按照4项子标题填写：1、指控的罪名；2、指控的犯罪事实；3、是否认罪认罚；4、量刑情节和量刑建议。
- 量刑情节项下的（1）至（4）是起诉书原文格式，保持原样。

**第三部分"受援人供述与辩解"**：
- **每次讯问笔录单独列出**（第一次、第二次、……），每次必须包含完整元数据（时间、地点、讯问人、记录人）。
- 摘录部分**直接复制原文**，不要改写或概括。
- **每次讯问笔录后必须跟【评价】**，不得遗漏。
- **公安阶段讯问笔录**：详细摘录，保留关键问答。
- **检察院阶段讯问笔录**：精简处理，只摘录与案件事实有关的内容，或存在争议的部分，其余简化。

**第四部分"同案犯供述与辩解"**：
- **只摘录与受援人定罪量刑有关的**同案犯供述，无关的略过。
- 不得将所有同案犯供述全部列出而不加筛选。

**第五部分"被害人情况及陈述"**：
- 有被害人陈述的，详细摘录。
- 无被害人陈述的，简要说明即可。

**第六部分"证人证言"**：
- **只摘录与受援人有关的**证人证言，无关的略过。
- 包含辨认笔录（含他人对受援人的辨认）。

**第七部分"客观性证据"**：
- 只列实际存在的证据类型，**没有的就不写**，不列空壳标题。
- **扣押情况不在此部分写**，统一放到第八部分，避免重复。

**第八部分"涉案财物扣押情况"**：
- 受援人涉案财物、涉案赃款追缴、同案犯涉案财物扣押，统一在此部分写。
- 注意不要与第七部分重复。

**第九部分"辩方证据"**：
- 已收集证据与待补充清单。

**第十部分"证据分析"**：
- 合法性、关联性、证明力、矛盾点、排除合理怀疑分析。

**第十一部分"相关法律规定"**：
- 法律规定要详细、全面，包括法律法规和司法解释。
- **必须进行幻觉审查**（验证法律引用的准确性），使用元典MCP工具或其他可靠来源。
- **不写"相关法律文书"**小节。
- **不写"综合评价"或"本部分综合评价"**。
- **不写"待核实问题"**。

**第十二部分"辩护思路"**：
- 只需包含辩护方向和具体辩护要点。
- **不写"庭审策略"**（发问提纲、质证要点、辩论意见等属于辩护词内容）。
- **不写"量刑建议"**（具体刑期建议属于辩护词内容）。

**落款**：
- 文档末尾必须有**阅卷人**和**阅卷时间**，不可遗漏。

### 通用撰写规则

- **模板严格遵守**：用户给定了格式模板的，必须一字不差地按模板结构填写，不得自作主张修改结构、字段名称或编号。
- **关联性筛选**：同案犯供述、证人证言等只摘录与受援人定罪量刑有关的内容，无关的略过，不写"未见直接涉及相关内容，故不摘录"。
- **格式一致性**：全文格式和风格必须保持一致（编号方式、页码标注、字体字号等）。
- **空壳标题**：没有内容的证据类型直接跳过，不列空标题也不写"无"。
- **部分间去重**：不同部分之间不要重复写同一内容（如扣押情况只在第八部分写）。
- 提取信息：人物信息、每次讯问元数据、核心供述、书证与客观证据清单。
- 合规复核：逐项检查归属、页码、时间线、程序要素完整性。
- 交付结果：优先提供结构化正文 + 风险点清单 + 待补证据事项。

## 会见笔录专项工作流（新增）

适用：用户要求“根据阅卷笔录填写会见笔录”，且明确保留模板内容。

1. 先读取会见笔录模板与阅卷笔录，列出争点清单（参与次数、角色分工、获利金额、时间区间、讯问合法性、认罪认罚过程、前科与累犯）。
2. 在不删除模板原有条目的前提下填充“律师端可预填内容”；“被会见人回答（答：）”默认留空。
3. 在“律师根据案情，自行提问”中填入不少于 5 个针对性问题，问题需直接对应争点清单。
4. 在“听取陈述后法律解答”位置补充个案化法律解释，避免一笔带过。
5. 在“本案风险及法律后果”中写明有利/不利量刑因素及程序风险，不使用空泛表述。
6. 在“探讨辩护方案、征询辩护意见”中明确辩护路径并征询是否同意。

## 会见笔录合规核对清单（新增）

### 笔录内容硬要求

- 表明身份并询问是否同意辩护。
- 告知法律援助无偿性质。
- 告知权利义务。
- 解释涉罪构成及量刑幅度，不得空白或模板化略写。
- 核实案件主要事实、情节、证据材料，问题应具体且可核验。
- 听取陈述/辩解后必须有针对性法律解答。
- 风险及法律后果必须与本案事实绑定。
- 必须沟通辩护方案并征询辩护意见。
- 认罪认罚案件须告知相关程序权利与法律后果。
- 审判阶段须补充诉讼程序、审判组织、回避权等说明。

### 笔录形式与现场事项

- 不删除模板原条目，仅做填充和补充。
- 会见时间、地点、会见人、记录人等抬头信息应完整。
- 空白项需规范处理（按本地办案规范划线或留白）。
- 每页签名/捺印/日期按归档规范执行。
- 涂改处按规范捺印或签注。

## 辨认笔录抽取与评价方法（新增）

### 辨认笔录完整性要求

阅卷笔录中辨认笔录应包含以下两类，缺一不可：

1. **受援人辨认笔录**（第3部分）：受援人对同案人员、赌场人员等的辨认
2. **他人对受援人的辨认**（第6部分）：同案人员、证人对受援人的辨认

**常见遗漏**：
- 只摘录受援人辨认他人，遗漏他人辨认受援人
- 遗漏被辨认人身份情况说明的页码标注
- 遗漏号位映射关系的明确表述

### 证据抽取分层

- 直接指认：辨认结果中明确出现”某号即为某人（含被告姓名）”。
- 间接指认：辨认结果仅出现”某号=某角色”，需再用”被辨认人身份情况说明”完成号位映射。
- 背景性材料：仅出现姓名于名单/身份页，但无辨认结果，不得写成”已被指认”。

### 写入规则

- 每段摘录必须标注卷别与页码。
- 对“号位映射”的结论使用“间接指认”表述，不得写成“直接辨认出某人”。
- 如 OCR 缺字、断句、页码错位，必须加“需核对原件”风险提示。
- 评价需写清证明方向：证明角色分工、参与程度、与供述是否互证、是否存在混号风险。

### 推荐写法（辨认证据）

```text
【X某辨认笔录】（诉讼证据卷第A页，身份说明见第B页）
【摘录】“……第N组第M号为外围望风人员……”
【对应关系】“第N组第M号=某某（身份说明页）”
【评价】该证据属于“号位指认+身份说明”的间接指认，可用于证明受援人角色；但需核验同批次照片编号一致性。
```

## 常见错误清单（输出前自检）

### 工作流错误
- [ ] **一次性生成全部内容**：必须逐部分撰写，逐部分确认，不得擅自跳到下一部分
- [ ] **擅自继续写下一部分**：写完一部分后必须等用户确认，不得自行续写

### 结构性错误
- [ ] 章节重复：”基本信息”与文档抬头重复，应从”一、基本情况”开始
- [ ] 章节顺序错误：”相关法律规定”必须是第 11 项，不是第 12 或 13 项
- [ ] 辨认笔录遗漏：只摘录受援人辨认他人，遗漏他人对受援人的辨认
- [ ] **不按模板格式填写**：自作主张修改结构、字段名称、合并或拆分子项
- [ ] **表头字段写多了**：只保留阅卷时间、提请审查起诉时间、阅卷地点、涉嫌罪名

### 内容性错误
- [ ] 供述归属错误
- [ ] 讯问字段缺失（时间/地点/人员/页码）
- [ ] 关键摘录不完整
- [ ] **缺少【评价】**：每次讯问笔录后必须跟评价
- [ ] **检察院阶段笔录过于详细**：只摘录与案件事实有关或存在争议的内容
- [ ] **同案犯/证人摘录缺乏筛选**：未关联受援人，把无关内容也写了
- [ ] 建议性语言过强，不符法律文书语体
- [ ] 模板提示语未删除
- [ ] 出生信息、前科、强制措施时间等基础事实与卷宗不一致
- [ ] 将”身份说明名单出现姓名”误写为”已被辨认人直接指认”
- [ ] 未区分”直接指认”与”间接指认（号位映射）”
- [ ] 会见笔录只填模板，未体现个案针对性问题和针对性法律解答
- [ ] **基本信息部分写了页码**：第一部分只介绍情况，不标注页码
- [ ] **到案经过写了扣押物品**：只写归案过程，不写扣押细节
- [ ] **第七部分和第八部分重复**：扣押情况只在第八部分写
- [ ] **空壳标题**：没有内容的证据类型仍列标题或写”无”
- [ ] **第11部分多余内容**：不应包含”相关法律文书””综合评价””待核实问题”
- [ ] **第12部分多余内容**：不应包含”庭审策略””量刑建议”

### 页码与格式错误
- [ ] **页码使用 OCR 行号**：如”诉讼证据卷第 63394 页”（这是 OCR 文本行号，不是卷宗页码）
- [ ] 被辨认人身份情况说明页码未标注或标注错误
- [ ] **字体错误**：Word 导出时全文应使用仿宋_GB2312，不是宋体或黑体
- [ ] **字号不分层级**：大标题（一、二、三、）应为二号（22pt），子标题（1、2、3、）应为小四号（12pt），正文为四号（14pt），三者不可混用
- [ ] **遗漏落款**：文档末尾必须有”阅卷人”和”阅卷时间”
- [ ] **编号格式混乱**：章节子标题使用（一）（二）（三）而非 1、2、3
- [ ] **Word 导出残留 Markdown 格式**：项目符号（小圆点）、分隔线（---）未清除
- [ ] **多余内容**：第12部分不应包含”庭审策略””量刑建议”；第11部分不应包含”相关法律文书””综合评价”
