# 刑事阅卷 B类（委托类）证据分析报告式模板

> 适用：B类（委托类）案件的阅卷笔录。本结构由《刑事案件全流程法律产品交付》中"阅卷笔录与证据分析报告"演化而来，作为 B类委托的**主模板**。
> A类（法律援助类）仍使用 SKILL.md 中的 12 部分结构，**不适用本模板**。

---

## 一、为什么 B类用报告式结构

- 委托类客户多为付费当事人/家属，更需要一份**论证导向、可读性强**的成品，能直接体现律师工作深度与价值。
- 报告式结构以"逐证三性分析表 + 证据链整体评估"为核心，比按证据种类平铺的 12 部分更聚焦辩点论证。
- **但形态变化不等于质量降级**：12 部分结构沉淀的全部硬质量护栏必须原样移植到本结构（见第四节）。

---

## 二、整体结构（8 部分）

```
封面/抬头
一、案件信息摘要
二、全案证据分类清单
三、逐证三性分析表（核心）
四、证据链整体评估
五、程序违法审查记录
六、辩护思路
落款
```

---

## 三、逐部分模板与撰写指引

### 封面/抬头

```
阅卷笔录（证据分析报告）

阅卷时间：YYYY年MM月DD日
提请审查起诉（或提请公诉）时间：YYYY年MM月DD日
阅卷地点：XX律师事务所
涉嫌罪名：XXXX
案号：（202X）X刑XX字第XX号（如已知）

委托人：XXX，男/女，XXXX年XX月XX日出生，身份证号码：XXXXXXXX
```

- 称谓用"委托人"或"当事人"，全文统一，禁止混入"受援人"。
- 阅卷地点通常为律所；委托手续未完善时标注"委托手续待补充"。

### 一、案件信息摘要

- 委托人基本信息（身份、家庭、身体、前科）
- 强制措施情况（种类、期限、起算日）
- 涉嫌罪名与法定刑期区间
- 案件基本事实简述（基于卷宗，客观陈述，不评价）

### 二、全案证据分类清单

按类别统计全案证据：
```
证据分类统计：
├─ 言词证据：XX份（委托人供述X份 / 同案犯供述X份 / 证人证言X份 / 被害人陈述X份）
├─ 实物证据：XX份（物证X份 / 书证X份 / 鉴定意见X份）
├─ 视听资料·电子数据：XX份
└─ 程序性文书：XX份
```
- 只统计实际存在的类型，没有的不列空壳。

### 三、逐证三性分析表（核心）

对每一份与定罪量刑相关的证据，逐条填入下表，并做**三性量化评分**：

| 编号 | 证据名称 | 证据类型 | 卷·页码 | 证明内容 | 真实性 | 合法性 | 关联性 | 综合评价 |
|------|---------|---------|--------|---------|-------|-------|-------|---------|
| 证1 | | | 诉卷第A-B页 | | ✅/⚠️/❌ | ✅/⚠️/❌ | ✅/⚠️/❌ | 采信/存疑/不采信 |

**三性评分规则**：
- ✅ 无异议　⚠️ 存疑（须在表下注明理由）　❌ 有重大问题（须注明理由+法律依据）
- 每个 ⚠️/❌ 必须附简要理由，**禁止只打符号不说理**。

**护栏注入（强制）**：
- **页码必须是真实卷宗页码**，严禁使用 OCR 文本行号（如 63394 这类五位数）。无法确定时写"卷内目录第X页"或"需核对原件"。
- **讯问类证据（供述/讯问笔录）**：在"证明内容"或表下备注中保留八要素——第几次讯问、起止时间（精确到分）、地点、讯问人（姓名+单位）、记录人（姓名+单位）、卷宗页码、关键问答摘录、【评价】。多次讯问逐次列。
- **供述归属校核**：每段供述对应正确陈述人，禁止张冠李戴；同案犯供述只取与委托人定罪量刑相关部分。
- **辨认笔录**：区分"直接指认"与"间接指认（号位映射）"，号位映射须附身份说明页页码，写"间接指认"不得写"直接辨认出某人"。
- **原件优先纠偏**：涉及签名、捺印、日期、印章的事实以可核验原件为准；OCR 未识别签名而原件可见签名的，记"原件签名已确认"，不得写成"签名缺失"。原件未回看的统一写"待核对原件"，不得把 OCR 空白直接转为不利事实。

### 四、证据链整体评估

- 有罪证据的证明强度评价（控方证据链是否完整）
- 无罪/罪轻证据的整理
- 证据链中的**断裂点、矛盾点**（关键事实冲突列明"冲突点—对应页码—可能影响"）
- 需要补充侦查的问题清单
- **排除合理怀疑分析**：全案证据能否形成完整锁链、能否排除其他可能性、孤证不能定案

### 五、程序违法审查记录

- 讯问过程合法性（有无同步录音录像、讯问时间地点是否合规）
- 取证程序合法性（有无非法证据线索）
- 鉴定程序合规性（机构/人员资质、检材来源、方法标准）
- 权利告知是否到位
- 发现的排非线索单独列出，供后续 criminal-trial-examination 制作排非申请使用

### 六、辩护思路

- 审查起诉阶段辩护方向与要点
- 审判阶段辩护方向与要点
- 待核实问题
- **不写"庭审策略"**（发问/质证/辩论属辩护词与庭审模块内容）
- **不写"量刑建议"**（具体刑期属辩护词内容）

### 落款

```
阅卷人：XXX
阅卷时间：YYYY年MM月DD日
```

## 四、强制移植的护栏清单（缺一不可）

B类报告式结构虽改变了组织形态，但以下护栏从 12 部分结构**全部移植，不得遗失**：

| 护栏 | 落点 |
|------|------|
| 原件优先纠偏规则 | 第三部分逐证表 + 第五部分程序审查 |
| 页码真实性（禁 OCR 行号） | 第三部分逐证表"卷·页码"列 |
| 讯问类证据八要素 | 第三部分讯问类证据备注 |
| 供述归属校核（禁张冠李戴） | 第三部分 |
| 辨认笔录直接/间接指认区分 | 第三部分 |
| 分步撰写、逐部分确认（不得一次性生成全部） | 执行流程 |
| 法条幻觉审查（引法条用元典MCP复验） | 涉及法律依据处 |
| Word 导出格式（仿宋_GB2312、字号分层、落款不可漏） | 导出环节 |
| 关联性筛选（同案犯/证人只摘与委托人相关） | 第三部分 |

---

## 五、分步撰写工作流（强制）

与 12 部分结构一致，B类报告式同样**逐部分撰写、逐部分确认**：
1. 读取材料（诉讼文书卷、证据卷、补充侦查卷）
2. 先出"封面+案件信息摘要+证据分类清单"，等律师确认
3. 再出"逐证三性分析表"（这是核心，篇幅最大，可分批确认）
4. 再出"证据链整体评估+程序违法审查记录"
5. 最后出"辩护思路+落款"
6. 不得在律师确认前擅自续写下一部分。

---

## 六、Word 导出格式

与 A类一致：
- 全文字体：仿宋_GB2312
- 大标题（"阅卷笔录（证据分析报告）"及"一、二、三…"）：二号（22pt）加粗居中
- 子标题（"1、2、3…"）：小四号（12pt）加粗
- 正文：四号（14pt）不加粗，1.5 倍行距，首行缩进 2 字符
- 表格（逐证三性分析表）：表头黑体加粗，内容仿宋
- 落款不可遗漏（阅卷人、阅卷时间）
- 清除 Markdown 残留（项目符号、分隔线）
